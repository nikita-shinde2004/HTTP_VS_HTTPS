🔑 SSL/TLS Concepts

Understanding how SSL/TLS secures HTTPS connections.

What is SSL/TLS?

SSL (Secure Sockets Layer) and its successor TLS (Transport Layer Security) are cryptographic protocols that provide secure communication over a network. When you see https:// and a padlock icon in your browser, TLS is working in the background.

Modern HTTPS uses TLS 1.2 or TLS 1.3. SSL is now deprecated due to known vulnerabilities.

TLS Handshake — Step by Step

  1. Client Hello — Browser sends: "I want to connect securely. Here are the cipher suites I support."
  2. Server Hello — Server replies: "Agreed. I'll use [cipher suite]. Here is my SSL Certificate."
  3. Certificate Verification — Browser verifies the certificate against trusted Certificate Authorities (CA).
  4. Key Exchange — Both sides securely generate and exchange session keys (using asymmetric encryption).
  5. Session Established — All data is now encrypted using the shared session key (symmetric encryption).
  6. Secure Communication — Encrypted data flows both ways.

Symmetric vs Asymmetric Encryption

TypeHow It WorksUsed ForExample
Asymmetric Public key encrypts, private key decrypts TLS Handshake (key exchange) RSA, ECDH
Symmetric Same key encrypts and decrypts Actual data transfer (fast) AES-256-GCM

SSL Certificate

An SSL certificate is a digital document issued by a trusted Certificate Authority (CA) that proves a website's identity. It contains the domain name, public key, CA signature, and expiry date.

The Three Security Properties of TLS

🔒 Confidentiality

Data is encrypted. Only the intended receiver can read it.

🛡️ Integrity

Data cannot be modified in transit. Any tampering is detected.

✅ Authentication

Certificates verify that you're talking to the real server, not an impostor.
