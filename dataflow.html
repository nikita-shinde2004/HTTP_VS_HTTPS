HTTP Communication Steps

  1. Client sends a plain-text request to the server.
  2. Request travels over the network without any encryption.
  3. Anyone on the same network (Wi-Fi, router) can intercept and read it.
  4. Server receives the request and responds (also unencrypted).

HTTPS Communication Steps

  1. Client initiates a TLS Handshake with the server.
  2. Server presents its SSL Certificate for identity verification.
  3. Both parties agree on encryption keys (session keys).
  4. All data is encrypted before being sent over the network.
  5. Even if intercepted, the data is completely unreadable (ciphertext).
  6. Server decrypts with the session key and processes the request.
